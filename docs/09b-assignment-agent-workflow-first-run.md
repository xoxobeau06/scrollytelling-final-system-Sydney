# Assignment: Agent Workflow First Run

**Assigned:** [Set in Canvas]  
**Due:** [Set in Canvas]

## Goal

Complete one full issue-driven agent cycle: improve one sub-issue with a coding agent, approve it, have the agent implement it, verify the result in the browser and console, and close the issue with a completion comment.

You are practicing the loop: **approve the plan, review the output.** The agent writes and edits code; you decide if the plan is good enough and if the result is correct.

## Why This Assignment

Issue-driven agent workflow is a real-world skill. Your job is not to type every line yourself. Your job is to evaluate the plan and judge the output. This assignment gives you one full cycle so you can use the same workflow on the rest of your Project 2 issues.

## Prerequisite

You must already have:

- Your Project 2 repo created (from the template or Start Here)
- Your master issue and all required sub-issues created and linked

You need at least **one open sub-issue** to use for this assignment (e.g. GSAP Motion Foundation).

## Steps

Follow the [Step-by-step first run](09a-agent-workflow-first-run.md) for full detail. Summary:

1. **Pick one open sub-issue** (e.g. GSAP Motion Foundation). Copy the issue URL or issue body.
2. **Ask your agent to improve the issue** (no coding yet). Use the "Improve this issue" prompt from [Working with Coding Agents](09-working-with-coding-agents.md). Paste your repo URL and the issue. When the agent returns improved issue text, copy it.
3. **Update the issue on GitHub** with the improved Goal, Definition of Done, Files/Links, Constraints, and Verification. Review using the four checks (scope, specificity, order, completeness). Approve or edit until the issue is ready to build from.
4. **Ask your agent to work the issue.** Use the "Work this issue" prompt from [Working with Coding Agents](09-working-with-coding-agents.md). Let the agent implement. Do not close the issue yet.
5. **Verify the work.** Open your project in the browser. Run the verification steps from the issue. Check the console. Read the diff. If something is wrong, fix it or leave the issue open.
6. **Add a completion comment and close the issue.** Use the format: What changed / How I verified it / Open risks or follow-ups / Ready for human review. Then close the issue.

## What To Submit In Canvas

- Link to the **closed** issue that you improved and had the agent implement
- The issue must contain a **completion comment** in the standard format (What changed / How I verified it / Open risks or follow-ups / Ready for human review)
- Optional: screenshot of the working result or the completion comment

## Grading (5 points, Complete/Incomplete style)

- 2 pts: One sub-issue was improved with an agent (improved issue text is visible in the issue body or in a comment)
- 2 pts: The agent implemented the issue; you verified in the browser and console; the work matches the issue
- 1 pt: The issue was closed with a completion comment that includes "What changed" and "How I verified it"

## If You Get Stuck

- Use [Working with Coding Agents](09-working-with-coding-agents.md) for the exact prompts and troubleshooting.
- If blocked for more than 20 minutes, open a **Blocker Report** issue and include what you tried and any error details.
